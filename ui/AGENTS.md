# AGENTS.md

This file provides directory-level guidance for agents working under `ui/`.

## Prototype Rule

If your task touches anything under `ui/prototype/`, you must read and follow:

- `ui/prototype/AGENTS.md`

Treat `ui/prototype/AGENTS.md` as the source of truth for:

- the purpose of the prototype
- the relationship between `ui/prototype/` and the real frontend project
- what may and may not be copied from the real frontend
- validation expectations for prototype changes

Do not make changes in `ui/prototype/` based only on general repo conventions. Always apply the prototype-specific rules from `ui/prototype/AGENTS.md` first.

## Scope

This file is intentionally minimal. It does not replace more specific instructions inside subdirectories.

When multiple `AGENTS.md` files apply, the more specific one wins.
