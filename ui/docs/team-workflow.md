# 团队工作流

需求 issue → design PR → code PR

## 设计 PR

内容必须包含：

- [ ] Pencil 文件（含设计稿高保真 & 交互说明）
- [ ] 演示 Demo
- [ ] 说明文档.md

## 开发 PR

- [ ] 已关联 Design PR
- [ ] 已确认使用 Design System 组件
- [ ] 无自定义 UI

### 工作流

* 对简单需求，设计师产出设计变更，（通过 AI Agent）产出代码变更并提交（研发 review & merge）
* 对复杂需求，设计师产出设计变更，提交 commit；研发基于设计变更 commit 产出代码变更

### 可能的工作

1. 设计稿是易于修改的（公共组件、design token）
2. AI 能理解前端代码跟设计稿的映射关系（目录结构相似、维护文档）
3. 前端代码本身是对 AI 友好的（tailwindcss、对 AI 友好的技术方案）

### 其他

* 基于当前状态得到一份设计稿作为以后设计工作的基础
  - AI 本地运行前端代码
  - AI 通过浏览器去访问本地站点，截图
  - AI 结合前端代码和截图，生成设计稿
  - 顺便生成一份文档用于指导 AI 以后理解设计稿跟前端代码的映射关系
